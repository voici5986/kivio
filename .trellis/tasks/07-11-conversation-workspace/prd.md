# 普通对话默认工作台

## Goal
为未绑定项目的普通对话提供稳定、可配置的默认工作台，使模型在用户未指定路径时，将文件产出和基础命令工作放到该对话独立目录中，同时不限制用户访问或使用其他路径。

## Background
当前 `workspace_roots` 仅部分影响 `run_command`，普通对话文件工具的相对路径与交付文件目录并不统一；生成文件仍使用独立的 `~/Kivio/outputs/<conversation-id>`。这导致模型不知道统一的默认工作位置，命令、文件操作和交付产出可能落在不同目录。

## Requirements
1. 设置中提供一个全局“默认工作目录”，默认值为 `<用户主目录>/Kivio/workspace`，兼容 Windows 与 macOS。
2. 设置支持文件夹选择、手动输入/粘贴和恢复默认。
3. 该设置仅作用于未绑定项目的普通对话；项目对话继续使用绑定的项目根目录。
4. 普通对话的默认工作台为 `<全局工作目录>/<对话ID>`。
5. 对话目录按需创建：首次调用文件工具、命令工具或产生交付文件时创建，不因仅创建对话而创建空目录。
6. 工作台是默认路径而非安全边界：
   - 相对路径和未指定 `cwd` 的命令默认从工作台解析/执行；
   - 用户明确给出的绝对路径、`~/` 路径或其他目录必须照常使用；
   - 不禁止读取、写入或执行工作台之外的路径。
7. 在模型上下文中明确告知当前工作台及上述优先级：用户指定位置优先，未指定时使用工作台。
8. 取消独立的 `outputs/<对话ID>` 目录；普通对话未指定位置的交付文件直接进入工作台，并继续显示为可下载文件卡片。
9. 删除普通对话时直接删除其对应工作台目录。删除前必须验证目标确实是当前配置根目录下由该对话 ID 派生的目录，禁止误删其他路径。
10. 修改全局工作目录时迁移已有普通对话目录；不得覆盖目标端同名文件。迁移失败时保留源文件并明确提示。
11. 旧 `workspace_roots` 配置自动迁移第一项为新全局工作目录；没有旧值时使用平台默认值。
12. 旧 `~/Kivio/outputs/<对话ID>` 内容迁移并合并进对应普通对话工作台，成功后不再保留旧目录。
13. 配置目录不存在时按需自动创建；创建失败时明确报错，不静默回退到其他目录。

## Non-goals
- 不增加对话级工作目录覆盖设置。
- 不把工作台做成沙箱或权限边界。
- 不改变项目对话的项目根目录语义。

## Acceptance Criteria
- 普通对话中未指定路径创建 PPT/文档/代码或运行命令时，默认位置为 `<全局工作目录>/<对话ID>`。
- 用户明确要求保存到桌面或其他绝对路径时，结果位于用户指定位置而非工作台。
- 项目对话相对路径仍从项目根目录解析。
- 文件工具、命令工具和交付文件使用同一普通对话工作台。
- 删除普通对话会删除对应工作台，但无法删除根目录外或非该对话派生的目录。
- 修改设置能安全迁移已有目录；冲突时不覆盖且源文件仍在。
- Windows 与 macOS 默认路径、路径选择和迁移逻辑通过测试。
- 旧配置与旧 outputs 数据能够迁移，升级后不再产生新的 outputs 对话目录。
