# 通用文件展示工具

## Goal

让 `present_artifacts` 成为简洁的通用文件展示工具，既能展示当前回复生成的 artifact，也能展示电脑上的现有文件。

## Requirements

- 工具支持 `artifact_ids` 和本地 `paths`，至少提供一项。
- 相对路径按当前工作目录解析，绝对路径按原路径解析。
- 本地路径必须指向真实文件，目录或不存在的路径应返回明确错误。
- 本地文件进入现有 artifact 展示链路，不新增另一套前端协议。
- 工具描述简短，不再限定“当前助手回复中生成的文件”。
- 已经返回 artifact ID 的文件优先使用 ID，避免重复读取路径。

## Acceptance Criteria

- [ ] 只传 artifact ID 时保持现有展示行为。
- [ ] 只传本地路径时可以展示对应图片或文件卡。
- [ ] artifact ID 与路径可混合使用，并保持输入顺序语义。
- [ ] 工具调用位置仍然决定文件在回复中的展示位置。
- [ ] 非法路径不会产生伪造展示记录。
- [ ] Rust 与前端相关测试通过。
