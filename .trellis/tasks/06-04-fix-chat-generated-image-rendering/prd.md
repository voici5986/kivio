# fix chat generated image rendering

## Goal

Fix the chat answer renderer so images generated by tool execution, such as Python-created PNG files, display normally in assistant responses instead of showing a broken image placeholder.

## What I Already Know

- The screenshot shows Python completed successfully and stdout reported `SUCCESS: jilin_weather.png generated ...`.
- The assistant response contains a broken image placeholder before the explanatory text.
- The likely failure is in the chat artifact / markdown image pipeline rather than Python generation itself.

## Requirements

- Inspect chat records, app logs, and relevant renderer/runtime code to identify where generated image references are lost or converted incorrectly.
- Preserve existing text rendering behavior.
- Render local generated image outputs in chat responses using a browser-loadable source.
- Avoid exposing arbitrary filesystem access beyond the app's existing local-resource safety model.

## Acceptance Criteria

- [ ] A tool-generated PNG referenced in chat output renders visibly in the chat answer area.
- [ ] Existing markdown text and non-image output continue to render.
- [ ] Frontend type-check and lint pass when practical.
- [ ] Backend tests or type checks are run if backend code changes.

## Out of Scope

- Redesigning the chat UI.
- Changing model/tool prompting behavior unless needed to preserve image references.
- Adding a new file manager or gallery.

## Technical Notes

- Start by inspecting chat message rendering, Python tool output handling, local file URL conversion, and any Tauri asset protocol helpers.
